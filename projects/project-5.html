Technical Deep Dive

ttyt is a Python-based CLI wrapper that uses advanced LLMs to bridge the gap between human intent and terminal commands. It features a three-tier safety classifier and autonomous task execution with intelligent error recovery.

Core Stack
  • Language: Python 3.8+ with type hints
  • AI Providers: Google Gemini, Z.ai (GLM-4), OpenRouter
  • CLI Framework: Rich for terminal UI and formatting
  • Platform: Cross-platform (Windows, Linux, macOS)
Key Features
  • Natural language to shell command translation
  • Three-tier safety system (Safe/Caution/Danger)
  • Autonomous agentic mode with auto-retry
  • Context-aware AI with command history
  • Multi-provider model switching
Safety Architecture

Every AI-generated command passes through a strict classifier:

  • SAFE: Read-only commands (ls, cat, git status) - Auto-executes immediately
  • CAUTION: Modifications (mkdir, npm install, git commit) - Requires confirmation
  • DANGER: Destructive operations (rm -rf, force push) - Blocked by default
Agentic Mode

The /agent command enables autonomous task execution with up to 3 iterations per goal:

  • Attempts command execution and analyzes failures
  • Explores errors with read-only commands (cat, grep, ls)
  • Generates fix commands based on error context
  • Verifies goal achievement via exit codes and output
