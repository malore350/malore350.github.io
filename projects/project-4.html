Unigo is a comprehensive university consultancy platform designed to help students find universities, manage applications, and schedule consultations.

The project is structured as a monorepo containing a high-performance web frontend, a mobile application, and a robust backend server with real-time features.

Technical Architecture

A modern monorepo architecture designed for scalability and seamless integration across web, mobile, and backend services.

1. Backend Server

  • Runtime & Framework: Node.js with Express.js.
  • Database & Auth: Supabase (PostgreSQL) with hybrid authentication (Supabase Auth + custom JWT).
  • Storage: Supabase Storage for secure document management.
  • Key Features: Automated background jobs for consultation management, real-time support chat, and AI-driven university suggestion engines.

2. Web Frontend

  • Framework: React (Vite) with TypeScript.
  • Styling: Tailwind CSS and Shadcn UI primitives for a consistent, accessible design system.
  • State Management: Zustand for lightweight, efficient global state.
  • Internationalization: Multi-language support via i18next.

3. Mobile App

  • Framework: React Native with Expo SDK 54.
  • Navigation: Expo Router for file-based routing.
  • UI Components: Custom design system with linear gradients and a warm, academic-focused palette.
  • Integration: Unified REST API communication using Axios.

User Types & Functionality

The platform operates with a robust role-based permission system, catering to different workflows for students, staff, and administrators.

Student (Regular User)

  • University Search: Manual browsing and AI-powered personalized recommendations.
  • Applications: Credit-based application system with real-time status tracking (Submitted, Conditional Offer, Full Acceptance).
  • Consultations: Integrated scheduling for video/audio sessions with counselors.
  • Management: Self-service document portal and live support chat.

Employee (Counselor/Staff)

  • Workflow: Manage assigned student applications and update lifecycle statuses.
  • Interaction: Direct student communication via chat and consultation management (scheduling/completion).
  • Operations: Specialized dashboard for tracking performance and processing student documents.

Admin (Superuser)

  • Platform Control: User role management, credit adjustments, and staff assignments.
  • Business Logic: Promo code creation, organization management, and financial overview (revenue/transactions).
  • Content: Full control over the master university database and accommodation listings.

Payment & Provisioning System

Unigo integrates the Epoint gateway (Azerbaijan) for a seamless financial experience, handling everything from subscription tiers to automated invoicing.

  • Monetization: Tiered subscription plans (LITE, PRO, MAX), 'a la carte' application credits, and AI-generation tokens.
  • Automated Workflow: Backend price calculation (including upgrade differentials), promo code validation, and secure transaction signing.
  • Fulfillment: Real-time signature verification, automated credit provisioning, and instant email delivery of contracts/invoices.
