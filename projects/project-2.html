Privacy-focused crypto platform with Tor network support.

A multi-currency trading platform that supports wallets, swaps, liquidity pools, and user management across a centralized backend, a Next.js frontend, and a React Native mobile app.

#Technical Overview

Built as a full-stack system with dedicated server, web, and mobile layers. The backend handles blockchain integrations, authentication, and data persistence, while the frontend focuses on responsive trading UX and real-time market displays.

  • Backend: Node.js + Express.js with JWT auth and bcryptjs.
  • Databases: MongoDB (Mongoose) and MySQL (Sequelize) via adapter pattern, switchable by USE_MYSQL.
  • Blockchain: Modules for BTC, ETH, XMR, ZEC, DASH, TRX; Binance API integration.
  • Services: node-cron jobs, Mailgun email, 2FA with speakeasy + qrcode.
  • Web: Next.js 15.1.6 (App Router), TypeScript, Tailwind CSS, shadcn/ui, next-themes.
  • Forms & Charts: react-hook-form + zod, recharts for market data.
  • Mobile: React Native + Expo SDK 54, React Navigation, secure token storage.

Blockchain Integration (Custodial Model)

The exchange uses a hybrid custodial architecture. A Node.js backend orchestrates wallet creation, deposit monitoring, and withdrawals while keeping user balances in the database for instant internal transfers.

  • On-chain deposits: Users deposit to generated addresses; the server detects confirmations and credits balances.
  • Off-chain transfers: Internal transfers are instant and zero-fee via DB updates (MongoDB/MySQL).
  • On-chain withdrawals: Withdrawals trigger real blockchain transactions from hot wallets.
Chain Technology Implementation Details
Bitcoin Direct RPC Connects to a local Bitcoin Core node via JSON-RPC and manages a custodial wallet.
Ethereum Web3.js + Alchemy Uses Alchemy RPC nodes for mainnet access, with gas and signing handled in Node.js.
Monero / Zcash RPC / Daemon Integrates with local daemon RPCs for privacy coins using chain-specific modules.
Tron / Dash Node.js Scripts Dedicated JavaScript modules handle address generation and withdrawals.

Operational & Security Controls

  • Market data: Binance API via binance-api-node, cached and proxied to the UI.
  • Hot wallets: Server-controlled hot wallets process withdrawals.
  • 2FA & PIN: High-risk actions protected by 2FA and secondary PIN flows.
  • Address management: Deposit addresses tracked per user and stored for reconciliation.

Privacy & Hosting

The project uses Tor hosting to support user privacy, censorship resistance, and anonymous access—common requirements in decentralized finance applications.
